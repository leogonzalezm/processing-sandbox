Compara cómo se expresan las mismas ideas visuales en DOM / CSS y en Processing / p5.js. Navega los 5 capítulos.

Lienzo DOM — 600 × 400 px
💡 Aquí aparecerán las formas. Cada elemento es un <div> posicionado con CSS.

Un punto necesita dos datos para existir: su posición horizontal (x) y su posición vertical (y). Estos datos se llaman argumentos: valores que le pasamos a una función para que sepa qué dibujar.

Comparación de código

DOM / CSS

/* Posicionar un elemento en el lienzo */
position: absolute;
left: 100px;  ← posición horizontal (x)
top:  100px;  ← posición vertical (y)

p5.js / Processing

point(100, 100);
//     ↑       ↑
// arg 1: x   arg 2: y
// horizontal  vertical
Mapa de argumentos — point(x, y)
ArgumentoQué controlaEn p5.jsEn DOM / CSS
x Posición horizontal (izquierda → derecha) 1.er argumento de point() left: x px
y Posición vertical (arriba → abajo) 2.do argumento de point() top: y px

Ideas clave

  • Un argumento es un dato que la función necesita para ejecutarse.
  • En p5.js los argumentos van entre paréntesis, separados por comas: point(x, y)
  • En CSS cada argumento es una propiedad separada: left para x, top para y.
  • El origen (0, 0) está en la esquina superior izquierda del lienzo.

Una primitiva es una forma visual base. rect() y triangle() necesitan argumentos de posición y tamaño. ellipse() se posiciona desde el centro. line() conecta dos puntos. Combina estas 4 primitivas para dibujar un paisaje.

Comparación de código

DOM / CSS

left:   260px;  ← x (posición)
top:    160px;  ← y (posición)
width:  80px;   ← w (ancho)
height: 80px;   ← h (alto)

p5.js / Processing

rect(260, 160, 80, 80);
//    ↑     ↑    ↑    ↑
//    x     y    w    h
//  posX  posY ancho alto
Mapa de argumentos por primitiva
Primitivap5.jsArgumentosEn DOM / CSS
rect rect(x, y, w, h) x, y = esquina; w, h = tamaño left, top, width, height
ellipse ellipse(x, y, w, h) x, y = centro; w, h = diámetros Igual + border-radius: 50%
triangle triangle(x1,y1, x2,y2, x3,y3) 3 vértices (6 valores) clip-path: polygon(...)
line line(x1, y1, x2, y2) Punto inicio + punto final Div delgado posicionado con left, top, width

Ideas clave

  • rect() posiciona desde la esquina superior izquierda de la caja.
  • ellipse() posiciona desde el centro (equivale a ellipseMode(CENTER)).
  • triangle() recibe 6 argumentos (3 vértices × 2 coordenadas). En CSS se logra con clip-path: polygon().
  • line() conecta dos puntos. En el DOM es un div delgado (3px de alto).
  • En el DOM un círculo es una caja con border-radius: 50%.
  • El orden de los argumentos importa: x, y, w, h — siempre.
  • Combina primitivas para crear paisajes: rectángulos para el suelo, triángulos para montañas, círculos para el sol, líneas para el horizonte.

El color en p5.js es un estado: llamas a fill(r, g, b) antes de dibujar y todo lo que venga después usa ese color. En CSS es un atributo del elemento: lo cambias directamente.

Elige un color → se aplica a todas las formas del lienzo.
Comparación de código

DOM / CSS

background-color: rgb(r, g, b);
/* Se aplica directamente al elemento */
/* Cambia un atributo del objeto existente */

p5.js / Processing

fill(r, g, b);   ← llamar ANTES
rect(x, y, w, h); ← dibuja con ese color
// fill() cambia el ESTADO gráfico
Mapa de argumentos — fill(r, g, b)
ArgumentoQué controlaRangoEn DOM / CSS
r Componente rojo 0 – 255 1.er valor de rgb(r, g, b)
g Componente verde 0 – 255 2.do valor de rgb(r, g, b)
b Componente azul 0 – 255 3.er valor de rgb(r, g, b)

Ideas clave

  • fill() no dibuja nada: solo cambia el estado del pincel.
  • En DOM cambias un atributo de un objeto ya existente.
  • En p5.js el orden importa: fill() debe ir antes del dibujo.

Una variable guarda un valor que puede cambiar. Los sliders convierten un número abstracto en una transformación visual. Mueve los sliders para ver cómo el dato controla la forma en tiempo real.

Crea un cuadrado o círculo primero. Luego vuelve aquí y mueve los sliders.
Comparación de código

DOM / JS

let w = slider.value;
elemento.style.width = w + "px";
// El slider alimenta la propiedad CSS
// La variable w conecta dato → visual

p5.js / Processing

let w = 80;
rect(x, y, w, h);
// w y h son variables, no números fijos
// Cambiar la variable = cambiar la forma
Variable → Argumento → Visual
VariableQué almacenaDónde se usaEfecto visual
w Ancho (valor del slider) 3.er arg. de rect(x,y,w,h) Cambia el ancho de la forma
h Alto (valor del slider) 4.to arg. de rect(x,y,w,h) Cambia el alto de la forma

Ideas clave

  • Una variable = nombre + valor que puede cambiar.
  • Los sliders hacen tangible la idea de dato dinámico.
  • Cuando una variable alimenta un argumento, el dibujo se vuelve interactivo.

Cada valor que usamos al programar tiene un tipo de dato. El tipo determina qué operaciones puedes hacer con él y cómo se comporta. En JavaScript y p5.js existen tipos primitivos (Number, String, Boolean, undefined, null) y tipos compuestos (Array, Object) que agrupan varios valores.

Escribe un valor y presiona "Evaluar tipo" para descubrir qué tipo de dato es.
Comparación de código

JavaScript / DOM

let dato = 42;
typeof dato; // "number"
// typeof devuelve el tipo como texto

p5.js / Processing

let dato = 42;
print(typeof dato); // "number"
// Mismo concepto, misma sintaxis
Tipos de datos — Tabla completa
TipoQué almacenaEjemplosUso en p5.js
Number Valores numéricos (enteros y decimales) 42, 3.14, -10, 0 Coordenadas, tamaños, colores: rect(10, 20, 80, 80)
String Texto (cadena de caracteres) "hola", "rojo", "px" Mostrar texto: text("hola", 50, 50)
Boolean Verdadero o falso (solo 2 valores) true, false Condiciones: if (mouseIsPressed)
undefined "Sin valor asignado" — valor por defecto de toda variable nueva let x; → x vale undefined Detectar si un argumento fue omitido: if (valor === undefined)
null "Vacío intencional" — el programador decide que no hay dato let img = null; Limpiar referencia: img = null; antes de cargar otra imagen
Array Lista ordenada de valores (de cualquier tipo) [10, 20, 30], ["a", "b"] Colecciones de partículas, colores, posiciones: vertices[i]
Object Pares clave-valor que agrupan datos relacionados { x: 10, y: 20 } Agrupar propiedades: particula.x, particula.y
Ejemplos por tipo — Primitivos

Number — opera matemáticamente

let x = 100;
let y = x + 50;  // 150
let mitad = x / 2; // 50
ellipse(x, y, mitad, mitad);

String — concatena texto

let nombre = "Hola";
let saludo = nombre + " mundo";
// "Hola mundo"
text(saludo, 50, 50);

Boolean — controla flujo

let visible = true;
if (visible) {
  rect(100, 100, 50, 50);
}

undefined — variable sin valor

let z;              // no asignamos nada
print(z);           // undefined
print(z + 10);     // NaN (no es número)
// Siempre asigna un valor inicial

null — vacío intencional

let img = null;     // aún no hay imagen
img = loadImage("foto.png");
// null indica "todavía no existe"
// Distinto de undefined ("olvidé asignar")

⚠ Error común: tipo incorrecto

let x = "100"; // ¡Es String, no Number!
let y = x + 50;  // "10050" (concatenó)
// Solución: Number("100") → 100
Ejemplos por tipo — Compuestos

Array — lista ordenada de valores

let colores = ["rojo", "verde", "azul"];
print(colores[0]);  // "rojo"
print(colores.length); // 3
// Los índices empiezan en 0

Array en p5.js — múltiples posiciones

let xs = [50, 150, 250];
for (let i = 0; i < xs.length; i++) {
  ellipse(xs[i], 200, 30, 30);
} // Dibuja 3 círculos

Object — agrupar datos relacionados

let particula = {
  x: 100,  y: 200,
  tam: 30, color: "#ff0"
};

Object en p5.js — usar propiedades

fill(particula.color);
ellipse(
  particula.x, particula.y,
  particula.tam, particula.tam
);
Processing (Java) vs p5.js (JavaScript) — Tipado

Processing (Java) — tipado estático

int x = 100;       // entero
float y = 3.14;    // decimal
boolean v = true; // verdadero/falso
String s = "hola"; // texto
color c = color(255, 0, 0);

p5.js (JavaScript) — tipado dinámico

let x = 100;       // Number (entero)
let y = 3.14;      // Number (decimal)
let v = true;      // Boolean
let s = "hola";    // String
let c = color(255, 0, 0);
Equivalencia de tipos: Processing (Java) → p5.js (JavaScript)
Processing (Java)p5.js (JavaScript)Diferencia clave
int Number Java distingue enteros de decimales; JS usa Number para ambos
float Number float almacena decimales; en JS todo número ya es decimal internamente
boolean Boolean Mismo concepto; en Java se escribe en minúscula (boolean)
String String Mismo concepto en ambos; Java usa solo "comillas dobles"
color p5.Color (objeto) En Java es un alias de int (ARGB); en JS es un objeto p5
void (funciones) No existe Java exige declarar qué retorna; JS infiere automáticamente

⚠ int vs float en Processing

int a = 7 / 2;    // 3 (¡trunca!)
float b = 7.0 / 2; // 3.5 (correcto)
// En Processing, int/int = int
// Usa float si necesitas decimales

En p5.js no hay este problema

let a = 7 / 2;    // 3.5 (siempre decimal)
let b = Math.floor(7 / 2); // 3
// JS siempre da decimal
// Usa floor() si quieres entero

Ideas clave

  • Number = datos para posiciones, tamaños y colores. Se puede operar matemáticamente.
  • String = texto entre comillas. El + con strings concatena en vez de sumar.
  • Boolean = true o false. Esencial para decisiones con if.
  • undefined = variable declarada pero sin valor. let x; → x es undefined.
  • null = vacío a propósito. El programador dice "aquí no hay nada aún".
  • Array = lista con índice numérico ([0], [1]…). Ideal para colecciones de formas o posiciones.
  • Object = pares clave:valor. Agrupa datos relacionados: particula.x, particula.y.
  • typeof te dice qué tipo tiene un valor: útil para depurar errores.
  • Mezclar tipos sin querer es un error común: "100" + 50 da "10050", no 150.
  • Processing (Java) usa tipado estático: debes escribir int, float, String al declarar. p5.js (JavaScript) usa tipado dinámico: solo escribes let y el tipo se deduce del valor.
  • En Processing, int / int = int (trunca decimales). Usa float para mantener precisión. En p5.js esto no ocurre.
