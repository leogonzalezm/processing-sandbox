Un programa interactivo no se ejecuta una sola vez de arriba hacia abajo. Necesita un ciclo de vida. En Processing, setup() prepara el entorno exactamente una vez y draw() funciona como un motor que dibuja un nuevo fotograma de manera continua, típicamente 60 veces por segundo.

Estructura mínima del sketch

Processing

void setup() {
  size(600, 400);
  background(232);
}

void draw() {
  ellipse(300, 200, 48, 48);
}

Qué significa

setup()
- Inicializa el lienzo y el estado base.
- Solo corre una vez.

draw()
- Repite el bloque completo.
- Cada repetición produce un fotograma nuevo.
Ciclo de vida del programa
EtapaCuándo ocurreQué debe ir ahíError común
setup() Solo al arrancar size(), configuraciones iniciales, tipografías, fondo fijo Poner animación aquí y esperar que cambie en el tiempo
draw() En bucle continuo Dibujo fotograma a fotograma, actualización de variables Olvidar que se repite y dibujar sin limpiar el fondo

Ideas clave

  • setup() prepara el sistema; draw() mantiene vivo el sketch.
  • La pantalla no es una hoja estática: es una secuencia de fotogramas.
  • Todo lo que pongas en draw() se vuelve potencialmente dinámico.

Cuando declaras size(ancho, alto), Processing guarda esos valores automáticamente en width y height. Eso permite hacer diseño relativo: ubicar elementos en el centro con width / 2 o cerca del borde inferior con height * 0.8 sin depender de números fijos.

Posicionamiento relativo con variables del sistema

Processing

void draw() {
  line(0, 320, 600, 320);
  ellipse(300, 200, 48, 48);
}

Versión relativa correcta

void draw() {
  line(0, height * 0.8, width, height * 0.8);
  ellipse(width / 2, height / 2, 48, 48);
}
Por qué conviene diseñar con width y height
ExpresiónQué produceVentajaEjemplo visual
width / 2 Centro horizontal El elemento sigue centrado aunque cambie el lienzo Sol, personaje, título
height / 2 Centro vertical Evita recalcular manualmente la mitad Objeto principal
height * 0.8 80% de la altura Sirve para suelo, horizonte o UI inferior Línea base, terreno

Ideas clave

  • width y height nacen automáticamente después de size().
  • Diseñar con proporciones hace que la composición no se rompa al redimensionar.
  • Los números fijos sirven para pruebas; las relaciones sirven para diseñar sistemas.

La animación básica aparece cuando una variable cambia dentro de draw(). Si una esfera usa una variable como posicionX y esa variable aumenta en cada fotograma, el ojo interpreta esa diferencia como movimiento. El lugar donde pongas background() cambia completamente el resultado visual.

Variable que se actualiza fotograma a fotograma

Animación limpia

float posicionX = 0;

void draw() {
  background(232);
  posicionX = posicionX + 2;
  ellipse(posicionX, height / 2, 48, 48);
}

Con rastro

float posicionX = 0;

void draw() {
  posicionX = posicionX + 2;
  ellipse(posicionX, height / 2, 48, 48);
  // Sin background(): quedan las huellas anteriores
}
Comparación visual del uso de background()
Ubicación de background()EfectoCuándo convieneQué enseña
Dentro de draw() Borra el fotograma anterior Animación limpia, juegos, interfaces La pantalla se redibuja completa cada frame
Fuera de draw() Acumula marcas y deja estela Arte generativo, trazas, dibujo procesual El tiempo puede usarse como material visual

Ideas clave

  • Animar es actualizar datos y volver a dibujar.
  • posicionX = posicionX + velocidad convierte una variable en movimiento.
  • background() no es decoración: controla si el tiempo se limpia o se acumula.

random() introduce decisiones no predefinidas: posiciones, tamaños, colores o ritmos generados por el computador. En Processing siempre devuelve un float. Si necesitas un entero, debes convertirlo de forma explícita; de lo contrario tendrás un error de tipo si intentas guardarlo en int directamente.

Aleatoriedad computacional y tipos cruzados

Uso correcto de random()

float x = random(width);
float tam = random(12, 48);
ellipse(x, random(height), tam, tam);

Error común con int

int columna = random(10);
// Error: random() devuelve float

int columnaOk = int(random(10));
// Conversión explícita a entero
Formas de usar random()
FunciónQué devuelveEjemploUso didáctico
random(max) Un valor entre 0 y max random(width) Posición horizontal aleatoria
random(min, max) Un valor entre min y max random(12, 48) Tamaño o velocidad variable
int(random(...)) Entero derivado de un float int(random(5)) Índices de array, columnas, grillas

Ideas clave

  • La aleatoriedad es útil cuando quieres delegar parte del diseño al sistema.
  • random() siempre devuelve decimal en Processing.
  • Si el programa necesita un entero, hay que convertir ese float de manera explícita.
Advertencia pedagógica: si tus estudiantes escriben int n = random(10);, no es un detalle menor: ahí aparece una diferencia concreta entre tipos de datos y valores de retorno. Conviene aprovechar ese error para reforzar qué es un float y por qué el lenguaje exige coherencia de tipos.
